External Source

Optionally sync your assets from a JSON endpoint instead of maintaining them manually.

The endpoint should return JSON containing either an assets array or an array of asset objects. Try https://localhost:3050/api/test-assets for sample data.

API Response Format

Your API endpoint should return JSON data in one of these formats:

Format 1: Simple Array

[
  {
    "name": "HQ - Seattle",
    "city": "Seattle", 
    "state": "WA",
    "ip": "10.1.1.100",
    "notes": "Global headquarters"
  },
  {
    "name": "Edge Node - Portland",
    "city": "Portland",
    "state": "OR",
    "ip": "10.2.1.50", 
    "notes": "Pacific Northwest edge site"
  }
]

Format 2: Object with Assets Array

{
  "title": "My Infrastructure Map",
  "assets": [
    {
      "name": "HQ - Seattle",
      "city": "Seattle",
      "state": "WA",
      "ip": "10.1.1.100", 
      "notes": "Global headquarters"
    }
  ]
}

Required Fields

  • name (or Name, hostname, Hostname) - Asset name or identifier
  • city (or City, location, Location) - City where the asset is located
  • state (or State, region, Region) - State/Province code (e.g., "CA", "TX", "ON")

Optional Fields

  • ip (or IP, ipAddress, IPAddress) - IP address of the asset (e.g., "10.1.1.100", "192.168.1.50")
  • notes (or Notes, description, Description) - Description or additional information
  • title - Map title (only used in Format 2)

Note: The API should return proper JSON with Content-Type: application/json header. Field names are case-insensitive and support common variations.
